Descriptions of Table Values

  • Date: The date of the alert.
  • Candid: This is a "candidate ID", and it is unique to an alert.
  • ObjectId: This is the "object ID", an ID for a specific astronomical object (a star, an asteroid...). Each objectId could exist multiple times, as there can be many candid for an objectId - for example, an object is observed on two nights, then it has two candid.
  • Jd: This is the precise time of the observation, given in Julian date.
  • Filter: This is the astronomical filter used. Astronomical observations are done through filters (like for photography, or like sunglasses are filters). ZTF is using the g,r,i filters.
  • Ra, Dec: These are the astronomical coordinates "Right ascension" and "Declination" from the Equatorial coordinate system.
  • Magpsf, Magap: These are astronomical magnitudes. In case you are not familiar with this: In astronomy, the brightness of an object (star, asteroid...) is measured in "magnitudes". The higher the magnitude, the fainter the object is; so, for example, 22 mag is very faint, 8 mag is quite bright.

Available Filters

  • You can filter the columns by entering values and then click the "Filter" button.
  • You can enter a specific value for the columns date, candid, objectId. You can also enter ranges in fields such as "magpf", Enter one of the following:
    1. >18
    2. <18
    3. 18.8 19.4
  • You can also filter the columns by clicking the sort indicators to sort in either ascending or descending order.
